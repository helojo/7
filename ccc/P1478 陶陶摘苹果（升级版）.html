    题目描述

    又是一年秋季时，陶陶家的苹果树结了n个果子。陶陶又跑去摘苹果，这次她有一个a公分的椅子。当他手够不着时，他会站到椅子上再试试。

    这次与NOIP 2005普及组第一题不同的是：陶陶之前搬凳子，力气只剩下s了。当然，每次摘苹果时都要用一定的力气。陶陶想知道在s<0之前最多能摘到多少个苹果。

    现在已知n个苹果到达地上的高度xi，椅子的高度a，陶陶手伸直的最大长度b，陶陶所剩的力气s，陶陶摘一个苹果需要的力气yi，求陶陶最多能摘到多少个苹果。

    输入格式

    第1行：两个数 苹果数n，力气s。

    第2行：两个数 椅子的高度a，陶陶手伸直的最大长度b。

    第3行~第3+n-1行：每行两个数 苹果高度xi，摘这个苹果需要的力气yi。

    输出格式

    只有一个整数，表示陶陶最多能摘到的苹果数。

    输入输出样例

    输入 #1

    8 15
20 130
120 3
150 2
110 7
180 1
50 8
200 0
140 3
120 2

    输出 #1

    4

    说明/提示

    所有数据：n<=5000 a<=50 b<=200 s<=1000

          xi<=280  yi<=100

    题目大意及分析

    题目意思很简单，其中 椅子的高度a，陶陶手伸直的最大长度b 这句话完全是为了丰富题目的故事性。实际处理中直接求和使用就好，没有什么特别的。本题用背包可以解，将陶陶遇到苹果摘与不摘 分为1,0，可以求解。但是这道题最佳的方法还是直接用贪心，直接开始摘最省力的苹果（谁不喜欢轻松一点嘛） 只要高度也符合。摘到没有力气摘下一个苹果为止。

    代码

    #include<iostream>
#include<algorithm>
using namespace std;

struct apple//结构体保存这个苹果的高度和所需要的力气；
{
    int h;
    int w;
};
bool compare(apple x,apple y)//比较函数升序 < ,降序 > ；
{
    return x.w<y.w;
}
int main()
{
    int n,s,a,b,sum=0,coun=0;
    cin >> n >> s;
    cin >> a >> b;
    apple p[n+1];
    for(int i=0;i<n;i++)
    {
        cin >> p[i].h >> p[i].w;
    }
    sort(p,p+n,compare);//将数组按照所花力气的大小升序排序；
    for(int i=0;i<n;i++)
    {
        if(sum+p[i].w>s)
            break;
        if(p[i].h<=a+b)
        {
            sum+=p[i].w;
            coun++;
        }
    }
    cout << coun << endl;
    return 0;
}
